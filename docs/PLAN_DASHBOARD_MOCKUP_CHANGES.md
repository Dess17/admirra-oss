# План переработки дашборда под макет (скрин 2)

## Сравнение: скрин 1 (сейчас) vs скрин 2 (макет)

---

## 1. ШАПКА / HEADER

### Сейчас (скрин 1)
- Лейбл: «ВЫБРАННЫЙ ПРОЕКТ» / «Общий дашборд»
- Заголовок: «Проект: 1» или название проекта
- Подзаголовок: зелёная точка + «АНАЛИТИКА И ПОКАЗАТЕЛИ ЭФФЕКТИВНОСТИ ПРО...»
- Справа: StatsFilters (Проект, Канал, Кампания, Период) + чекбокс «Учитывать НДС»
- Иконка ChartBar в синем квадрате

### Макет (скрин 2)
- Верхняя строка (мелкий текст): «Общая аналитика по всем активным проектам»
- Основной заголовок: «Отчет по проекту: Детейлинг Иркутск»
- Справа: дропдауны «Все каналы», «2 недели» (без Проект/Кампания в шапке?)
- Кнопки PDF/Telegram (уже убраны по запросу)

### Изменения для шапки
| № | Элемент | Действие |
|---|---------|----------|
| 1.1 | Верхний лейбл | Заменить «ВЫБРАННЫЙ ПРОЕКТ» на «Общая аналитика по всем активным проектам» (или аналогичный контекстный текст) |
| 1.2 | Заголовок | Формат «Отчет по проекту: {название}» вместо «Проект: 1» |
| 1.3 | Подзаголовок | Упростить или убрать зелёную точку; текст по макету |
| 1.4 | Фильтры в шапке | Оставить только «Канал» и «Период» в шапке (если в макете так); Проект и Кампания — вынести или оставить в другом месте |
| 1.5 | Иконка | Проверить наличие/стиль иконки в макете |

---

## 2. ГРАФИК «Эффективность кампаний»

### Сейчас (скрин 1)
- **Тип:** мультилинейный (6 линий: Расход, Показы, Переходы, Лиды, CPC, CPA)
- **Стиль:** линии без заливки, разные цвета (оранжевый, синий, зелёный, красный, фиолетовый, розовый)
- **Оси Y:** несколько осей (скрыты при 3+ метриках), значения в K/M
- **Ось X:** даты (19 Feb – 05 Mar)
- **Тултип:** тёмный, все 6 метрик в одном тултипе
- **Легенда:** под графиком, цветные кружки + названия метрик
- **Высота:** h-80 (320px)

### Макет (скрин 2)
- **Тип:** одна линия с заливкой (area chart)
- **Стиль:** синяя линия + градиентная заливка под линией (светло-синий → насыщенный синий)
- **Ось Y:** одна ось, подписи 0, 200, 400, 600, 800, 1.0K, 1.3K
- **Ось X:** даты (11 Feb – 21 Feb)
- **Тултип:** один показатель (например «64.3664») в тёмно-синем прямоугольнике
- **Легенда:** в макете не видна (возможно скрыта или одна метрика)
- **Расположение:** график слева, сайдбар справа (бок о бок)

### Изменения для графика
| № | Элемент | Действие |
|---|---------|----------|
| 2.1 | Тип графика | Перейти с мультилинии на одну линию с заливкой (area chart) |
| 2.2 | Метрика | Решить, какую метрику показывать по умолчанию (Расход? Показы? или выбор пользователя) |
| 2.3 | Заливка | Добавить градиент под линией: от прозрачного/светло-синего к синему |
| 2.4 | Цвет линии | Синий (#3b82f6 или по макету) |
| 2.5 | Ось Y | Одна ось, явные подписи (0, 200, 400, 600, 800, 1.0K, 1.3K) |
| 2.6 | Тултип | Упростить: один показатель, тёмно-синий фон, без списка всех метрик |
| 2.7 | Легенда | Убрать или заменить на селектор метрики (если одна линия) |
| 2.8 | Layout | График в левой колонке, сайдбар справа (см. раздел 4) |

## 3. KPI-КАРТОЧКИ (ОБЩАЯ СТАТИСТИКА)

### Сейчас (скрин 1)
- **Сетка:** 6 колонок в ряд (grid-cols-6)
- **Подпись:** «Расходы за эту неделю», «Показы за эту неделю» и т.д.
- **Тренд:** только процент (⇩ 300.0%, ⇡ 628.0%)
- **Иконки:** синие кружки (dollar, eye, arrow, paw, hand, square)
- **Расположение:** под графиком, перед таблицей кампаний

### Макет (скрин 2)
- **Сетка:** 2×3 (2 колонки, 3 ряда)
- **Подписи:** «Расходы» + «За период»; «Показы» + «По всем каналам»; «Клики» + «Все переходы»; «CPC» + «Стоимость клика»; «Лиды» + «По всем каналам»; «CPA» + «Стоимость лида»
- **Тренд:** процент и абсолютное изменение («+15.6%», «+1.4k за эту неделю»)
- **Иконки:** синие (wallet, bar chart, pointer, currency tag, list, checkmark)
- **Расположение:** внизу страницы, ниже графика и сайдбара

### Изменения для KPI
| № | Элемент | Действие |
|---|---------|----------|
| 3.1 | Сетка | 6 колонок → 2 колонки (2×3) |
| 3.2 | Подписи | Двухуровневые: заголовок + подпись («За период», «По всем каналам» и т.д.) |
| 3.3 | Тренд | Добавить абсолютное изменение («+1.4k за эту неделю», «-213 за эту неделю») |
| 3.4 | Иконки | Заменить на wallet, bar chart, pointer, currency tag, list, checkmark (по макету) |
| 3.5 | Позиция | Переместить KPI вниз (после графика+сайдбара и таблицы кампаний?) |

## 5. ПОДКЛЮЧЕННЫЕ КАНАЛЫ (Сайдбар)

### Сейчас (скрин 1)
- Заголовок + кнопка «Добавить»
- Список: иконка, название, статус («Активно» / «Не подключено»)
- Баланс справа (если подключено)
- Нет переключателей (toggle)

### Макет (скрин 2)
- Заголовок + кнопка «Добавить +» (с плюсом)
- Список: иконка (цветная — жёлтая Y, синяя VK, красная шестерёнка), название, баланс («10 200P», «9 900P»), toggle (зелёный когда активно)
- Три канала: Yandex Direct, VK Ads Manager, «Шаблон канала»

### Изменения для каналов
| № | Элемент | Действие |
|---|---------|----------|
| 5.1 | Кнопка | «Добавить» → «Добавить +» |
| 5.2 | Toggle | Добавить переключатель (Switch) для каждого канала |
| 5.3 | Баланс | Формат «10 200P» (без пробела перед P или с пробелом — по макету) |
| 5.4 | Иконки | Цветные иконки (Yandex — жёлтая, VK — синяя) |
| 5.5 | Шаблон канала | Добавить «Шаблон канала» в список (если нужно по макету) |

## 7. ДОПОЛНИТЕЛЬНО

### Backend / данные
- Абсолютное изменение тренда («+1.4k за эту неделю») — нужно считать на бэкенде или на фронте
- Расписание отчётов — нужен API и хранение настроек

### Компоненты для правок
- `StatisticsChart.vue` — основной объём изменений (график)
- `GeneralStats3.vue` — layout, порядок блоков
- `KPIOverview.vue` + `CardV3.vue` — KPI-карточки
- `StatsHeader.vue` — шапка
- `ConnectedChannelsV3.vue` — каналы
- `ReportSendingBlock.vue` — отправка отчётов

## Порядок реализации (рекомендуемый)

1. **Layout** — изменить порядок: график + сайдбар бок о бок, KPI внизу
2. **График** — переделать на area chart с одной метрикой
3. **KPI** — сетка 2×3, подписи, тренд с абсолютным изменением
4. **Шапка** — текст и структура по макету
5. **Подключенные каналы** — toggle, кнопка «Добавить +», формат баланса
6. **Отправка отчётов** — расписание, кнопка «Сохранить», визуал с галочками (без Bitrix24/Trello)
