# Анализ: Лиды vs Конверсии и отображение целей

## Проблема 1: Расхождение Лидов и Конверсий на 2

### Описание
- **Лиды** (KPI-карточка) и **Конверсии** (воронка) обычно показывают одно значение (79).
- Иногда возникает разница в 2 (например, Лиды = 79, круговая диаграмма = 77 шт).

### Источники данных

| Элемент UI | Источник | API/метод |
|------------|----------|-----------|
| Лиды (KPI) | `summary.leads` | `StatsService.aggregate_summary` → `get_summary` |
| Конверсии (воронка) | `summary.leads` | Тот же `summary` — строка 224 в PromotionEfficiency.vue |
| Круговая диаграмма (центр) | `totalConversions` | Сумма `goal.count` из `GET /dashboard/goals` |

### Цепочка данных

**Лиды / Конверсии в воронке:**
- `StatsService.aggregate_summary` → `m_q` = `SUM(MetrikaGoals.conversion_count)` WHERE `goal_id != "all"`
- Фильтр по `client_ids`, `date`, при выборе кампаний — по `integration_ids`

**Круговая диаграмма:**
- `GET /api/dashboard/goals` (`stats.get_goals`)
- Запрос: `SUM(MetrikaGoals.conversion_count)` GROUP BY `goal_id`, `goal_name` WHERE `goal_id != "all"`
- Фильтр: `client_id`, `date_from`, `date_to`; опционально `integration_id` (дашборд его не передаёт)

### Возможные причины расхождения на 2

1. **Кеширование**
   - `get_goals` имеет `@cache_response(ttl=900)` (15 мин).
   - `get_summary` кешируется отдельно (CacheService).
   - При обновлении данных один источник может отдать свежие данные (79), другой — закешированные (77).

2. **Разные периоды**
   - Summary: `filters.start_date`, `filters.end_date`.
   - Goals: `date_from`, `date_to` из PromotionEfficiency.
   - Оба получают даты из GeneralStats3, но при смене периода возможна рассинхронизация (например, goals ещё с прошлого периода).

3. **Фильтр по интеграциям**
   - При выборе кампаний `aggregate_summary` фильтрует MetrikaGoals по `integration_ids` выбранных кампаний.
   - `get_goals` вызывается без `integration_id` и возвращает цели по всем интеграциям клиента.
   - В этом случае сумма в диаграмме может быть больше, а не меньше. Для обратной ситуации (79 vs 77) более вероятен кеш.

4. **Fallback на goal_id="all"**
   - При отсутствии индивидуальных целей `get_goals` использует `goal_id="all"`.
   - `aggregate_summary` всегда исключает `goal_id="all"`.
   - В этом случае возможна разная логика подсчёта.

### Рекомендации по исправлению (Проблема 1)

1. Синхронизировать инвалидацию кеша: при обновлении данных целей очищать кеш и summary.
2. Передавать в `get_goals` те же фильтры, что и в summary (campaign_ids → integration_ids), чтобы источники совпадали.
3. Добавить логирование: при расхождении `summary.leads` и `sum(goal.count)` писать в лог для отладки.

## Проблема 2: Все цели отображаются как «Автоцель»

### Описание
- В настройках выбраны все цели **кроме** автоцелей (скрин 2).
- На дашборде все цели показаны с префиксом «Автоцель:» (скрин 3).

### Причина

**Файл:** `admin-panel-vue-main/.../PromotionEfficiency.vue`  
**Функция:** `formatGoalName` (строки 244–246)

```javascript
const formatGoalName = (name) => {
  if (!name) return 'Автоцель'
  return name.startsWith('Автоцель:') ? name : `Автоцель: ${name}`
}
```

Логика: для любой цели без префикса «Автоцель:» он добавляется принудительно. В результате:
- «Все заявки» → «Автоцель: Все заявки»
- «Переход с поиска» → «Автоцель: Переход с поиска»
- «Автоцель: клик по номеру» → остаётся без изменений

То есть все цели визуально превращаются в «автоцели», независимо от типа в Метрике.

### Откуда берутся имена

- `get_goals` возвращает `goal_name` из `MetrikaGoals.goal_name`.
- При синхронизации (`sync.py`) имя берётся из `goal_names_map` (API Метрики).
- Метрика возвращает:
  - автоцели: `"Автоцель: клик по номеру телефона"`, `"Автоцель: заполнил контактные данные"` и т.п.;
  - обычные цели: `"Все заявки"`, `"Переход с поиска"`, `"Marquiz - общий"` и т.п. — без префикса.

### Рекомендации по исправлению (Проблема 2)

1. **Убрать принудительное добавление префикса** — показывать имя цели как есть:
   ```javascript
   const formatGoalName = (name) => name || 'Цель'
   ```

2. **Либо** определять автоцели по типу цели (если API отдаёт тип) и добавлять «Автоцель:» только для них. Сейчас `get_goals` не возвращает тип цели, поэтому надёжнее опираться на имя из API.
