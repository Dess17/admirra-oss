# Настройки проекта телефонии — что реально работает

При создании/редактировании проекта часть настроек срабатывает только при наличии соответствующих переменных окружения или внешних сервисов.

---

## Выгрузка

| Настройка | Работает? | Что нужно |
|-----------|-----------|-----------|
| **CRM Webhook URL** | Да | Достаточно указать URL в форме. После валидации лида система шлёт POST с телом в JSON на этот URL. Доп. env не нужны. |
| **Email получатели** | Да, если настроен SMTP | В `.env`: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM` (и при необходимости `SMTP_USE_TLS`). Иначе отправка почты отключена. |
| **Telegram Chat ID** | Да, если настроен бот | В `.env`: `TELEGRAM_BOT_TOKEN`. В поле проекта — **chat_id того чата, куда бот будет присылать уведомления** (ваш личный чат с ботом или чат группы). |

### Про Telegram: кому пишет бот

Бот **не пишет лиду (клиенту)**. Он пишет **вам (или группе)** — в тот чат, чей **chat_id** вы указали в настройках проекта.

- Лиду писать первым не нужно.
- Вам нужно один раз: написать боту в Telegram (или добавить бота в группу), получить **chat_id** этого чата и подставить его в поле «Telegram Chat ID» проекта.
- Как получить chat_id: написать боту сообщение, затем открыть `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` — в ответе будет `"chat":{"id": 123456789}`. Это и есть chat_id.

Итого: сообщения о новых заявках приходят в указанный вами чат; лид с ботом общаться не обязан.

## Валидация

| Настройка | Работает? | Что нужно |
|-----------|-----------|-----------|
| **Проверка соцсетей (TT, WA, МАКС, VK)** | Да, если включён хотя бы один источник | В `.env` один из вариантов: `VK_API_TOKEN` (VK), или `GETCONTACT_API_KEY` + `GETCONTACT_API_URL`, или `NUMBUSTER_API_KEY` + `NUMBUSTER_API_URL`. Если ни один не задан — проверка соцсетей не выполняется. |
| **Проверка Госуслуг** | Реализовано в коде | Зависит от наличия и настроек сервиса проверки Госуслуг в конфиге. |
| **Отправка в Яндекс.Метрику** | Да, если включена Метрика | **Вариант 1 (рекомендуется):** привязать проект к клиенту и подключить интеграцию «Яндекс» в разделе интеграций — тогда для отправки офлайн-конверсий используется уже сохранённый OAuth-токен и счётчик этой интеграции. **Вариант 2:** в `.env`: `METRICA_ENABLED=true`, `METRICA_COUNTER_ID`, `METRICA_OAUTH_TOKEN`. |

## Итог

- **Без доп. настроек** из коробки работают: **CRM Webhook** (достаточно указать URL в проекте).
- **Почта** — после настройки SMTP в env.
- **Telegram** — после создания бота, получения chat_id и указания его в проекте (бот пишет вам/группе, не лиду).
- **Соцсети** — после настройки VK и/или GetContact/NumBuster в env.
- **Метрика** — через интеграцию Яндекса (привязка проекта к клиенту + подключённая интеграция) или через счётчик и OAuth в env.

## Ссылки на сервисы (проверка соцсетей и API)

| Сервис | Переменные окружения | Где взять ключи / документация |
|--------|----------------------|---------------------------------|
| **VK** | `VK_API_TOKEN` | **Важно:** метод `users.search` доступен только с **пользовательским OAuth-токеном** (ключ доступа пользователя). Сервисный ключ даёт ошибку 1051. Получить: 1) создайте Standalone-приложение на [dev.vk.com](https://dev.vk.com); 2) откройте в браузере `https://oauth.vk.com/authorize?client_id=ВАШ_APP_ID&scope=offline&redirect_uri=https://oauth.vk.com/blank.html&response_type=token`; 3) скопируйте `access_token` из URL после редиректа. |
| **GetContact** | `GETCONTACT_API_KEY`, `GETCONTACT_API_URL` | Официального публичного API для разработчиков нет. Сервис [getcontact.com](https://getcontact.com/ru/), корпоративные/партнёрские интеграции — через поддержку или партнёрские программы. |
| **NumBuster** | `NUMBUSTER_API_KEY`, `NUMBUSTER_API_URL` | [NumBuster](https://numbuster.com/ru/) — корпоративные решения и интеграция по API для бизнеса. Ключи и URL получают по запросу (сайт, поддержка). |
